    class langtorch.TextModule

    TextModule

    TextModule is the base class for constructing text-based neural network modules within the LangTorch framework. It is intricately designed to work with TextTensor objects, enabling structured text manipulation and processing through the integration of prompt-driven transformations and optional language model (LLM) calls.

    In LangTorch, models designed for handling text data should subclass TextModule. This approach not only extends torch.nn.Module's functionality to text processing but also facilitates the creation of complex, nested module architectures. Developers can encapsulate specific text processing tasks within submodules, which are then hierarchically organized by assigning them as regular attributes, forming a tree-like structure.

    Parameter Type Description
    prompt TextTensor The ID of the language model to use for processing.
    activation TextModule or str Predefined system message to guide the model's response.
    key str Number of completions to generate for the input, adds a dimension.

    Example: Text Model

    To illustrate, consider defining a TextModule subclass that incorporates a sequence of text transformation tasks. Each task is represented by a TextModule configured with its unique prompt and LLM activation:

    import torch
from langtorch import TextModule, OpenAI

llm = OpenAI("gpt-3.5-turbo", "You're an assistant.", 0.5)

class TextTransformationModel(TextModule):
    def __init__(self):
        super().__init__()
        self.translate = TextModule("Translate to English: {}", activation=llm, key="text")
        self.summarize = TextModule("Summarize this text: '{text}'", activation=llm)

    def forward(self, input_text):
        translated_text = self.translate(input_text)
        return self.summarize(translated_text)

    The module translates and summarizes text using a shared activation.

    Note

    Ensure super().__init__() is called before adding submodules for correct initialization.

    This TextTransformationModel performs text translation followed by summarization. Each processing step is modularized by a TextModule, specifying the task through its prompt and the language model for activation.

    Assigning submodules in this structured manner ensures their registration within the LangTorch framework, which is crucial for comprehensive management of parameters and states. This includes ensuring compatibility with operations like .to(), facilitating device management, and parameter optimization.

    Note

    It's imperative to call super().__init__() before submodule assignment in the subclass. This ensures proper initialization and registration of the module within LangTorch, enabling the effective use of nested TextModule instances. Such a setup allows for the development of sophisticated text processing pipelines, leveraging the dynamic and structured nature of TextTensor for advanced NLP tasks.

    Following the initial introduction and example usage of TextModule, let's continue with further details and functionalities that TextModule offers within the LangTorch framework.

    Sequential and Parallel Processing

    TextModule inherently supports both sequential and parallel processing of TextTensor inputs. This functionality is crucial for optimizing performance, especially when dealing with batched inputs or multiple prompts that necessitate parallel processing through language model activations.

    Sequential Processing with torch.nn.Sequential

    LangTorch's TextModule can be seamlessly integrated into PyTorch's sequential models. This allows for the straightforward chaining of multiple text processing steps, each represented as a TextModule, within a unified processing pipeline.

    import torch
from langtorch import TextModule, Activation

# Define a sequential processing pipeline
pipeline = torch.nn.Sequential(
    TextModule("Translate this text to English: {}", activation=Activation(model="gpt-3.5-turbo", system_message="You're a translator.", T=0.5)),
    TextModule("Summarize the translated text: {}", activation=Activation(model="gpt-3.5-turbo", system_message="You're a summarizer.", T=0.5))
)

# Example input TextTensor
input_text_tensor = TextTensor(["This is a text to be translated and summarized."])

# Execute the processing pipeline
output = pipeline(input_text_tensor)

    This example showcases how to construct a processing pipeline that first translates text to English and then summarizes the translated text. Each step in the pipeline is modularized by TextModule, facilitating clear and efficient text processing flows.

    Warning

    Ensure that the TextTensor inputs match the expected format of the prompt templates accurately. If there is a mismatch between the placeholder name used in the module prompt and the corresponding key in the input tensor the completion text will be appended at the end of the text instead of being placed at the placeholder.

    Note

    • TextModule can be extended through subclassing to implement custom forward methods, offering tailored text processing capabilities. Some currently available subclasses are listed in langotrch.tt
    • Parallel (batched) processing of inputs through the LLM activation is handled automatically for all entries regardless of shape.
